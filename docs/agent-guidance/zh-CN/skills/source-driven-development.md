# Source-Driven Development

## 何时使用

- 使用或升级 Electron、Vue、Vite、TipTap、SQLite、测试框架、构建工具。
- 对框架 API、版本行为、安全默认值、弃用项做判断。
- 处理浏览器/Electron/Web API、Node 文件系统、SQLite FTS 等可能随版本变化的内容。

## 原则

不要只凭记忆或训练数据决定现代工具链行为。优先查：

1. 本仓库代码和锁文件。
2. 官方文档。
3. 官方迁移指南、release note、API reference。
4. 依赖源码或类型定义。

如果无法确认，必须标记为“未验证”，不要把猜测写成事实。

## 流程

1. 确认当前版本

   从 `package.json`、`pnpm-lock.yaml`、源码 import 和配置文件确认真实版本。

2. 查官方来源

   优先使用官方文档或项目源码。引用具体链接或文件位置。

3. 对照本仓库实现

   不要因为文档说“推荐”就立刻重构；先看现有架构、项目阶段和设计文档。

4. 记录结论

   关键框架/API 决策写入设计文档或代码注释，特别是 Electron 安全、SQLite 迁移、TipTap 存储格式。

## 验收清单

- [ ] 已确认实际依赖版本。
- [ ] 框架/API 结论来自官方或本仓库证据。
- [ ] 不确定处已标记，不伪装成确定事实。
- [ ] 结论和本地优先、安全边界不冲突。
