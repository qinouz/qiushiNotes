# 表格笔记

## 解决的问题

用户需要在笔记应用里直接维护类似有道云笔记的表格，典型场景是写日报、任务清单、简单台账。当前富文本表格能力不足以承担接近 Excel 的交互，例如单元格选区、复制粘贴、行列操作、手动调整列宽和行高。

## 目标

- 新增一种笔记内容格式：`spreadsheet-json`。
- 新建“表格笔记”后，右侧编辑区直接显示可编辑的电子表格。
- 表格支持基础单元格输入、复制粘贴、增删行列、常用格式和工作表底部标签。
- 必须支持用户手动调整列宽和行高。
- 表格变更写入本地 SQLite 的 `notes.content`，保持本地优先和自动保存。
- 搜索可以检索表格内的普通单元格文本。
- 默认模板满足日报场景：日期、任务项、优先级、状态、备注。

## 非目标

- 第一版不做 `.xlsx` 导入导出。
- 第一版不要求公式能力；如果 Univer 核心包自带公式能力，不作为当前验收重点。
- 第一版不做多人协作、在线同步、权限控制、模板市场。
- 第一版不把表格数据拆成独立 SQLite 业务表，避免破坏后续迁移到更完整表格引擎的可能性。

## 开源方案参考

- TriliumNext 已经提供 Spreadsheet note type，说明“笔记类型 + 表格引擎快照”是可行路线。
- Univer Sheets 是开源表格 SDK，官方文档说明它支持浏览器、Electron，并提供工作簿快照、行列增删、行高列宽设置等 API。
- Luckysheet 已不再适合作为新功能基础，官方仓库推荐迁移到 Univer。

本项目第一版采用 Univer Sheets Core Preset。使用第三方表格引擎承接复杂交互，本项目只负责笔记类型、生命周期、本地保存、搜索提取和未来导出迁移边界。

## 用户流程

1. 用户点击“新建”菜单。
2. 选择“表格笔记”。
3. 系统创建一条 `spreadsheet-json` 笔记，默认标题为“未命名笔记”，内容为日报表格模板。
4. 用户在右侧表格中编辑单元格，拖动表头边界调整列宽，拖动行头边界调整行高。
5. 表格命令执行后触发防抖保存，保存状态复用现有笔记自动保存提示。
6. 用户切换笔记或关闭应用前，当前表格快照会尽量刷新到草稿内容。

## UI 行为

- 左侧树和文件夹视图使用表格图标区分表格笔记。
- 编辑器顶部仍保留标题输入、保存状态和删除按钮。
- 表格区域占满右侧剩余空间。
- Univer 负责表格工具栏、公式栏、网格、选区、工作表底部标签等基础 UI。
- 公式栏可作为输入栏使用，但第一版不宣传公式能力。

## 数据模型

`notes.content_format` 新增：

```text
spreadsheet-json
```

`notes.content` 保存 JSON 包装对象：

```json
{
  "schemaVersion": 1,
  "engine": "univer",
  "engineVersion": "0.25.0",
  "savedByFacade": true,
  "workbook": {}
}
```

`workbook` 是 Univer 的 `IWorkbookData` 快照。包装层用于未来区分引擎版本、迁移策略和可能的导出元数据。`savedByFacade` 表示该快照来自 Univer Facade API 的 `workbook.save()`，旧的手工初始化快照会在打开时迁移为 Facade 保存格式。

## 本地存储行为

- 表格数据仍然只写本地 SQLite。
- 自动保存沿用现有 `notes:update` IPC，不新增后端服务。
- `note_search_index` 是派生数据；表格保存后由服务层重新抽取单元格文本并更新索引。
- JSON 导出直接包含 `spreadsheet-json` 的完整内容，保证用户可迁移。

## IPC / API 边界

- Renderer 不直接访问文件系统或 SQLite。
- 表格编辑器只通过现有 `update:content` 事件把快照交给笔记保存链。
- Main 进程只负责创建默认表格内容、规范化 `content_format`、提取搜索文本。

## 同步影响

同步未实现前，`spreadsheet-json` 与其他笔记一样通过 `version` 和 `sync_status` 标记本地变更。未来同步时应把整个 workbook 快照作为笔记正文同步；如果出现冲突，应生成冲突副本，而不是合并覆盖单元格。

## 错误处理和边界

- 表格 JSON 解析失败时，编辑器创建一个新的空日报模板，避免用户看到空白崩溃页。
- 旧版手工拼出的 workbook 快照如果导致渲染不稳定，打开时先用 Univer 默认工作簿初始化，再把已有单元格文本、列宽、行高迁移进去。
- 保存快照失败时，保留当前 Univer 实例，显示现有保存失败状态。
- 切换笔记时先触发当前草稿刷新，降低未保存风险。
- 如果 Univer 初始化失败，页面显示错误信息，原始内容仍保存在 SQLite 中。

## 隐私和数据安全

- 表格内容不上传服务器。
- 不使用远程 CDN 加载 Univer 资源。
- CSP 只在 `img-src` 放行 `data:`，用于 Univer 内置 SVG 纹理和画布图案；脚本仍然只允许本地资源。
- 不在表格快照中写入用户本机绝对路径。
- 表格快照随本地备份、JSON 导出一起保留。

## 测试和验证计划

- `pnpm.cmd typecheck`
- `pnpm.cmd --filter desktop build`
- `pnpm.cmd --filter desktop smoke:spreadsheet`
- `git diff --check`
- 自动化 smoke：
  - 使用隔离的 `QIUSHI_USER_DATA_DIR` 启动 Electron 构建产物，避免写入真实用户数据。
  - 通过 Electron `--remote-debugging-port` 连接 renderer，只在 `QIUSHI_SMOKE=1` 时启用测试钩子。
  - 覆盖新建表格笔记、写入单元格、触发快照、等待外层自动保存、切换到其他笔记再切回、搜索单元格文字。
  - smoke 钩子只操作当前已经挂载的 Univer workbook，不绕过 `update:content` 和现有 `notes:update` 自动保存链路。
- 手动验证：
  - 新建表格笔记后出现日报模板。
  - 输入单元格后保存状态变化并回到已保存。
  - 切换笔记再切回，单元格内容仍存在。
  - 拖动列宽和行高后保存，切换回来仍保持尺寸。
  - 搜索单元格文字可以找到该表格笔记。
