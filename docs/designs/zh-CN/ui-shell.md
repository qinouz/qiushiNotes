# UI Shell

## 状态

草稿

## 背景

当前界面已经有基础三栏：

```text
左侧：窄功能栏
中间：搜索、新建、笔记本/笔记文件树
右侧：编辑器
```

用户参考了有道云笔记的界面：左侧功能导航、搜索、新建按钮、笔记本树和大编辑区。我们不在这一阶段完整复刻视觉，但需要先把 UI shell 的组件边界拆清楚，后续才能逐步调整。

## 目标

- `App.vue` 只负责应用入口。
- `MainLayout.vue` 只负责布局组合。
- 左侧只保留窄功能导航，把“笔记、收藏、标签、回收站、设置”等长期入口放在固定位置。
- 中间栏承载搜索、新建和文件树，文件树同时显示笔记本和笔记。
- 为后续搜索、笔记本树、回收站、标签、收藏等能力留位置。
- 当前功能行为不变。

## 非目标

- 不实现有道云笔记完整视觉复刻。
- 不引入复杂路由系统。
- 不引入 Pinia。
- 不在左侧功能栏直接展示大量笔记本，避免窄导航失去稳定性。

## 当前组件结构

```text
renderer/
  App.vue
  layouts/
    MainLayout.vue
    FunctionBar.vue
    MiddlePane.vue
  features/
    notebooks/
      NoteTree.vue
      useNoteTree.ts
    notes/
      NoteEditor.vue
```

职责：

- `App.vue`：应用入口。
- `MainLayout.vue`：组合窄功能栏、中间文件树、编辑器。
- `FunctionBar.vue`：展示稳定的功能入口，V1 只开放“笔记”。
- `MiddlePane.vue`：展示搜索框、新建按钮和文件树。
- `NoteTree.vue`：展示笔记本与笔记的统一树形结构。
- `NoteEditor.vue`：展示标题、正文和保存状态。
- `useNoteTree.ts`：管理笔记本、笔记树、选中节点、创建、重命名和自动保存。

## 当前 UI 方向

采用接近有道云笔记的工作区结构：

```text
窄功能栏：最新、笔记、收藏、标签、待办、设置等长期入口
中间栏：搜索、新建、笔记本/笔记文件树
编辑区：标题、工具栏、正文
```

布局约束：

- 左侧功能栏宽度保持在约 64px，只负责稳定入口。
- 中间栏宽度保持在约 250px，避免压缩编辑区。
- 中间栏不再拆出独立笔记列表，笔记直接作为文件树节点显示。
- 点击文件夹用于选中新建目标并展开/折叠；点击笔记用于打开编辑器。
- 顶部“新建”默认在当前选中文件夹内新建笔记；没有选中文件夹时进入默认笔记本。
- 搜索框先做本地已加载树节点过滤，后续全文搜索模块完成后再接 SQLite FTS。

但第一版优先保证：

- 打开快。
- 编辑稳定。
- 自动保存可靠。
- 数据结构清晰。

## Electron 学习点

UI shell 属于 renderer 层。它只负责组织组件和响应用户操作，不应该直接访问：

- SQLite。
- 本地文件系统。
- Electron main 进程模块。

所有本地能力仍然通过：

```text
renderer -> preload -> IPC -> main
```

## 测试计划

- 类型检查通过。
- desktop 构建通过。
- 新建、编辑、自动保存、删除功能不受影响。
- 左侧功能栏收窄后，中间文件树和右侧编辑器不重叠。
- 点击文件夹后可以在该文件夹下新建笔记。
- 点击笔记后可以打开、编辑并自动保存。
- 搜索树节点时不会修改本地数据。
