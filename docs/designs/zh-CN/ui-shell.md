# UI Shell

## 状态

草稿

## 背景

当前界面已经有基础三栏：

```text
左侧：笔记本占位
中间：笔记列表
右侧：编辑器
```

用户参考了有道云笔记的界面：左侧功能导航、搜索、新建按钮、笔记本树和大编辑区。我们不在这一阶段完整复刻视觉，但需要先把 UI shell 的组件边界拆清楚，后续才能逐步调整。

## 目标

- `App.vue` 只负责应用入口。
- `MainLayout.vue` 只负责布局组合。
- 左侧笔记本区域拆成独立组件。
- 为后续搜索、笔记本树、回收站、标签、收藏等能力留位置。
- 当前功能行为不变。

## 非目标

- 不实现真实多级笔记本 CRUD。
- 不实现搜索框功能。
- 不实现有道云笔记完整视觉复刻。
- 不引入复杂路由系统。
- 不引入 Pinia。

## 当前组件结构

```text
renderer/
  App.vue
  layouts/
    MainLayout.vue
  features/
    notebooks/
      NotebookSidebar.vue
    notes/
      NoteList.vue
      NoteEditor.vue
      useNotes.ts
```

职责：

- `App.vue`：应用入口。
- `MainLayout.vue`：组合左侧笔记本区、笔记列表、编辑器。
- `NotebookSidebar.vue`：展示应用名、笔记本入口和占位导航。
- `NoteList.vue`：展示笔记列表和新建按钮。
- `NoteEditor.vue`：展示标题、正文和保存状态。
- `useNotes.ts`：管理笔记状态和自动保存。

## 未来 UI 方向

可以逐步接近有道云笔记的工作区结构：

```text
窄功能栏：最新、笔记、收藏、标签、待办、设置
侧边栏：搜索、新建、笔记本树
内容区：笔记列表或笔记本内条目
编辑区：标题、工具栏、正文
```

但第一版优先保证：

- 打开快。
- 编辑稳定。
- 自动保存可靠。
- 数据结构清晰。

## Electron 学习点

UI shell 属于 renderer 层。它只负责组织组件和响应用户操作，不应该直接访问：

- SQLite。
- 本地文件系统。
- Electron main 进程模块。

所有本地能力仍然通过：

```text
renderer -> preload -> IPC -> main
```
