# 功能设计文档

本目录存放中文设计文档。以后开发任何功能模块之前，先写或更新这里的设计文档，再开始实现代码。

设计文档的目的不是写形式主义材料，而是让以后回头看某个功能时，能快速知道：

- 当时为什么要做这个功能
- 功能边界是什么
- 数据怎么存
- UI 怎么交互
- 和本地优先、同步、导出有什么关系
- 代码实现有哪些关键取舍

推荐文件名：

```text
功能名.md
```

当前基础文档：

```text
template.md
community-reference.md
project-structure.md
local-database.md
```

新增功能时，可以从 `template.md` 复制一份作为起点。
