# 回收站与恢复

## 状态

草稿

## 背景

秋实笔记 V1 已经在笔记删除时写入 `deleted_at`，但如果没有回收站入口，用户无法确认删除结果，也无法把误删笔记恢复回来。

回收站是本地优先笔记应用的数据安全闭环之一。它的第一目标不是清理空间，而是让用户敢于删除、也能从误操作中回来。

## 目标

- 展示已软删除的笔记。
- 支持把已删除笔记恢复到普通笔记列表。
- 恢复时保留标题、正文、富文本格式、图片和附件引用。
- 恢复操作只写本地 SQLite，不依赖服务器。
- 继续维护 `version` 和 `sync_status`，为未来同步删除/恢复状态做准备。

## 非目标

- 暂不实现清空回收站。
- 暂不实现单条彻底删除。
- 暂不删除或清理附件文件。
- 暂不实现已删除笔记的完整编辑。
- 暂不实现笔记本回收站。

## 用户流程

1. 用户在笔记树或编辑器中删除笔记。
2. 笔记从普通笔记列表消失，并进入回收站。
3. 用户点击左侧功能栏“回收站”。
4. 应用展示已删除笔记列表，包含标题、摘要、删除时间和原笔记本信息。
5. 用户选择某条笔记并点击恢复。
6. 应用清空该笔记的 `deleted_at`，刷新回收站列表。
7. 用户回到“笔记”视图后，可以在普通笔记列表中再次看到该笔记。

## UI 行为

- 功能栏开放“回收站”入口。
- 回收站页面使用独立视图，不和普通笔记树混在一起，避免误把已删除内容当成正常内容编辑。
- 回收站为空时显示空状态。
- 加载失败时显示错误信息和可重试入口。
- 恢复按钮需要在操作中禁用，避免重复提交。
- 恢复成功后留在回收站页面，并从列表移除已恢复笔记。

## 数据模型

复用 `notes` 表：

```text
id
title
content
content_format
notebook_id
created_at
updated_at
deleted_at
version
sync_status
```

回收站列表只查询 `deleted_at IS NOT NULL` 的笔记。

恢复时更新：

```text
deleted_at = NULL
updated_at = now
version = version + 1
sync_status = 'pending'
```

如果原 `notebook_id` 对应的笔记本不存在或已删除，恢复时把笔记放回默认笔记本。这样用户能在普通列表里找到它，而不是恢复到一个不可见位置。

## 本地存储

- 回收站状态只存储在 SQLite 的 `deleted_at` 字段中。
- 附件文件继续保留在 `attachments/` 目录。
- 删除笔记不会物理删除附件，恢复后图片和附件引用应继续可用。

## IPC / API 边界

Renderer 通过 preload 暴露的受控 API 调用：

```ts
window.qiushi.notes.listTrashed()
window.qiushi.notes.restore(id)
```

Main 进程 IPC：

```text
notes:list-trashed
notes:restore
```

Service 层负责：

- 查询已删除笔记。
- 校验恢复目标是否仍处于删除状态。
- 处理原笔记本不可用时的默认笔记本回退。
- 维护 `version` 和 `sync_status`。

## 同步影响

V1 不实现远程同步，但恢复仍然标记 `sync_status = 'pending'`。

未来同步时：

- 删除可以作为带 `deleted_at` 的更新同步到服务器。
- 恢复可以作为 `deleted_at = NULL` 的更新同步到服务器。
- 如果服务器和本地同时修改，应保留冲突副本，而不是覆盖用户恢复出的内容。

## 异常处理

- 要恢复的笔记不存在：提示“笔记不存在或已被恢复”。
- 要恢复的笔记 `deleted_at` 已为空：视为已经恢复，提示用户刷新。
- 默认笔记本创建失败：恢复失败，不修改原笔记。
- SQLite 更新失败：保留原删除状态，并显示错误。
- 附件文件缺失：不阻止恢复笔记正文，但后续附件视图应提示文件缺失。

## 隐私与数据安全

- 回收站不上传任何内容。
- 回收站列表只展示本地已删除笔记。
- 不在日志中记录完整笔记正文。
- 不开放彻底删除，直到备份、导出和用户确认流程足够清楚。

## 测试计划

- 手动删除一条普通笔记，确认普通列表消失、回收站可见。
- 手动恢复已删除笔记，确认普通列表重新可见、正文内容保留。
- 删除包含图片的笔记后恢复，确认图片仍能显示。
- 连续恢复同一条笔记时，第二次不会造成重复记录。
- 运行 `pnpm.cmd typecheck`。
- 运行 `pnpm.cmd --filter desktop build`。
