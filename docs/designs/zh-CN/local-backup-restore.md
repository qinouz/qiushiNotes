# 本地备份与恢复设计

## 状态

草稿，作为云同步前置能力优先实现。

## 背景

秋实笔记是本地优先应用，本地 SQLite 和附件目录是数据源头。在服务器同步稳定之前，用户仍然需要一种可靠方式把公司电脑的数据带到家里电脑继续使用。

本模块先提供手动备份和手动恢复。它不是完整云同步，但能验证数据可迁移、可恢复，也能作为后续同步和升级前的安全保险。

## 目标

- 用户可以在应用内创建一个可复制的备份文件。
- 备份文件包含 `notes.db` 一致性快照、附件目录和清单文件 `manifest.json`。
- 备份默认保存到 Electron `userData/backups` 目录。
- 备份文件使用 `.tar.gz`，避免引入额外压缩依赖。
- 用户可以从应用内选择 `.tar.gz` 备份文件恢复。
- 恢复前先对当前数据创建安全备份，避免误操作直接覆盖。
- 恢复完成后重启应用，使 SQLite 连接重新指向恢复后的数据库。

## 非目标

- 不做云端上传、账号、设备列表或自动后台同步。
- 不做增量备份；每次备份都是完整备份。
- 不做附件去重和断点续传。
- 不在恢复时合并两端数据；恢复是整包替换。
- 不把备份密码、加密和密钥管理放入本次实现。

## 用户流程

### 创建备份

1. 用户打开设置页。
2. 点击“创建备份”。
3. main 进程使用 SQLite `VACUUM INTO` 生成一致性的 `notes.db` 快照。
4. main 进程把数据库快照、附件目录和清单文件打成 `.tar.gz`。
5. renderer 显示备份文件路径、大小和创建时间。
6. 用户可以把该文件复制到另一台电脑。

### 恢复备份

1. 用户在另一台电脑打开设置页。
2. 点击“从备份恢复”。
3. main 进程弹出文件选择框，只选择 `.tar.gz` 备份文件。
4. main 进程解包到临时目录并校验清单文件。
5. main 进程先为当前数据创建安全备份。
6. main 进程关闭当前 SQLite 连接，替换 `notes.db` 和附件目录。
7. main 进程重启应用。

## UI 行为

- 设置页显示“数据备份”区域。
- “创建备份”按钮执行完整备份。
- “从备份恢复”按钮执行文件选择和恢复。
- 恢复前 renderer 使用确认弹窗提示这是整包替换。
- 操作中按钮禁用，避免重复触发。
- 成功创建备份后显示备份路径和大小。
- 恢复开始后应用会重启，重启前不继续让用户编辑。

## 数据模型

本次不新增 SQLite 表。

备份包内部结构：

```text
manifest.json
notes.db
attachments/
```

清单文件字段：

```ts
{
  app: 'qiushi-notes'
  version: 1
  createdAt: string
  databaseFile: 'notes.db'
  attachmentsDir: 'attachments'
}
```

## 本地存储行为

- 备份输出到 `userData/backups/qiushi-notes-backup-<timestamp>.tar.gz`。
- 备份过程的临时文件放在 `userData/backups/.tmp-<id>`。
- 临时文件成功或失败后都应清理。
- 恢复前安全备份也放在 `userData/backups`。
- 替换数据库时删除旧的 `notes.db-wal` 和 `notes.db-shm`，避免 WAL 文件与恢复后的数据库不匹配。

## IPC / API 边界

preload API：

```ts
window.qiushi.backups.create()
window.qiushi.backups.restoreFromFile()
```

IPC 通道：

```text
backups:create
backups:restore-from-file
```

renderer 不接触真实文件路径写入逻辑，只展示 main 返回的结果。

## 同步影响

- 本模块是云同步前置能力。
- 恢复是整包替换，不做同步合并。
- 后续同步引擎上线前，可要求用户先创建本地备份。
- 后续冲突处理不能依赖本模块，但可以复用备份安全策略。

## 错误处理和边界

- SQLite 快照失败时不生成备份包。
- 附件目录不存在时创建空附件目录写入备份。
- 备份包清单文件缺失或 app/version 不匹配时拒绝恢复。
- 解包时拒绝包含 `..` 或绝对路径的条目，避免路径穿越。
- 恢复失败时保留当前数据，不删除安全备份。
- 如果应用无法重启，应至少返回错误并保留当前数据。

## 隐私和数据安全

- 备份文件包含全部本地笔记和附件，属于敏感数据。
- 第一版备份不加密，用户需要自行保管。
- UI 文案必须明确提示备份文件包含私有数据。
- 不自动上传备份文件到网络位置。

## 测试和验证计划

- 类型检查通过。
- 构建通过。
- 创建备份后生成 `.tar.gz` 文件。
- 备份包中包含 `manifest.json`、`notes.db` 和 `attachments` 目录。
- 恢复时选择无效文件会失败，不覆盖当前数据。
- 恢复有效备份后应用重启。
- 中文 UI 和文档无乱码。
